Smart Greenhouse Controller (soil moisture, light, humidity, temperature), IoT based system

Project Introduction

This project is an IoT‑enabled smart greenhouse controller system built around an ESP32 microcontroller. It automates plant care by monitoring and controlling key environmental parameters such as temperature, humidity, soil moisture, and light. The system combines sensor data, automation logic, and cloud connectivity to simplify indoor farming and improve plant growth.

This report demonstrates how to integrate sensors, actuators, and an IoT platform (Blynk) into a complete, remotely monitored greenhouse solution.

Smart greenhouse garden

1. Project Overview & Challenges

The goal of the system is to maintain optimal growing conditions automatically, while still allowing remote monitoring and manual control. During development, several engineering challenges were addressed:

  • Sensor calibration: Achieving accurate and consistent readings from DHT22, soil moisture, and light sensors.
  • Power management: Designing for 24/7 operation, including battery‑powered use.
  • Data synchronization: Keeping ESP32 and Blynk cloud data in sync despite intermittent connectivity.
  • Component integration: Combining sensors, pumps, fans, relays, and lights into a cohesive system.
  • Environmental robustness: Handling changing light, temperature, and humidity conditions inside the greenhouse.

2. Key Components

Core hardware and software elements:

  • ESP32 Board: Central microcontroller for data processing and WiFi connectivity
  • DHT22 Sensor: Measures temperature and humidity
  • Soil Moisture Sensor: Detects dry/wet soil conditions
  • Photoresistor: Monitors ambient light levels
  • Submersible Pump: Automated irrigation control
  • 2 × 5 V Mini Fans: Temperature and humidity control
  • 4‑Channel Relay Module: Switches pump, fans, and grow lights
  • LED Grow Lights: Provides artificial light for plant growth
  • Lithium Batteries: Enables off‑grid operation
  • Blynk IoT Platform: Remote monitoring and control dashboard

3. System Architecture

The ESP32 reads sensor data and applies automation rules to control actuators. Real‑time data and control commands are synchronized with the Blynk cloud, allowing remote access from a smartphone or web dashboard.

High‑level flow:

  • Sensors → ESP32 → Automation Logic → Relays → Pump/Fans/Lights
  • ESP32 ↔ Blynk Cloud → Mobile Dashboard (monitoring + manual override)

4. Automation Logic

The greenhouse uses simple threshold‑based rules to maintain suitable conditions:

  • Watering: Pump activates when soil moisture < 28% (or ADC threshold).
  • Cooling: Fan 1 turns on when temperature > 25 °C.
  • Humidity Control: Fan 2 runs when humidity > 65%.
  • Lighting: LED grow lights activate when ambient light < 50 lux (light‑intensity threshold).

5. Programming Core

The ESP32 firmware periodically reads sensors, updates the Blynk dashboard, and applies control logic. Below is a simplified example of the sensor reading and control routine: 

// Sensor Reading samples
void readSensors() {
    humidity    = dht.readHumidity();
    temperature = dht.readTemperature();
    moisture    = analogRead(SOIL_SENSOR);

    if (moisture < 300) {
        activatePump();
    }
    if (temperature > 28) {
        activateFan1();
    }
    // Additional logic for the humidity, light, etc.
}

6. Remote Monitoring Interface

Using the Blynk IoT platform, a custom dashboard was created to visualize data and control the greenhouse remotely.

  • Live temperature and humidity gauges
  • Soil moisture percentage display
  • Manual override switches for pump, fans, and lights
  • Status indicators for automation events
