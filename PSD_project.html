Arduino + ADXL335 Vibration Measurement and Power Spectral Density (PSD) Analysis

1. Arduino to ADXL335 Wiring Connections

The ADXL335 accelerometer operates at 3.3V, and its three analog outputs (X, Y, Z) are routed directly to the Arduino’s ADC inputs for sampling and processing.

2. Serial Communication Interface

A dedicated LabVIEW serial communication module initializes the COM port, applies the selected baud rate, and continuously reads incoming ADC data from the Arduino. The Analog Read VI retrieves raw voltage values for all three axes, which are then passed into the signal conditioning pipeline.

3. Signal Conditioning and Acceleration Conversion

Each axis undergoes two key conditioning steps:

  • Voltage Bias Removal: The ADXL335 outputs a mid‑supply bias (1.65 V) at 0 g. This offset is subtracted from the measured voltage.
  • Sensitivity Scaling: Using the sensor sensitivity (≈300 mV/g), acceleration is computed as:
a(g) = (V_measured − V_bias) / sensitivity

Calibrated acceleration samples are appended to arrays using shift registers, forming time‑domain datasets for each axis.

4. Spectral Measurement (PSD) Processing

Three identical PSD modules process the X, Y, and Z acceleration arrays. Each module computes:

  • Frequency vector
  • PSD amplitude
  • Averaged spectral results

The PSD reveals dominant vibration frequencies and provides insight into system dynamics across the 1–1000 Hz range.

5. Execution Management and Controls

The system includes three independent stop mechanisms:

  • Run Timer: Defines total acquisition duration based on sampling rate and user‑selected run time.
  • Main Loop Stop: Terminates data acquisition and serial communication.
  • Spectral Averaging Stop: Halts PSD processing while allowing continued data collection.

6. Results

Time‑Domain Acceleration: The system produced stable tri‑axial acceleration readings with expected noise characteristics over a 3‑minute run.

Power Spectral Density: PSD plots showed clear frequency peaks and accurate spectral distribution across all axes, confirming the system’s ability to capture and analyze vibration signatures.

7. Conclusion

The developed system successfully integrates Arduino‑based data acquisition with LabVIEW signal processing to deliver real‑time vibration monitoring and PSD analysis. It provides reliable acceleration measurements, calibrated outputs, and meaningful frequency‑domain insights suitable for vibration diagnostics and structural monitoring.
